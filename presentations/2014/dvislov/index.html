Потоковая передача данных

  • передаём файл в gulp, результат сохраняем в другой файл
  • нет промежуточных файлов и папок

grunt

gulp

Gulp plugin

  • каждый плагин должен выполнять только одно простое действие
  • gulp всего лишь соединяет и организует эти действия в задачи
  • нет дублирующих и конфликтующих плагинов

gulp API

				.src(globs[, options])
				.dest(path)
				.task(name[, deps], fn)
				.run(tasks...[, cb])
				.watch(glob [, opts], cb)

Gruntfile

Configuration over code

Gulpfile

Code over configuration

gruntfile example

    		jade:
		      compile:
		        options:
		          pretty: true
        files: [
          cwd: "app/jade"
          src: [ "**/*.jade", "!**/_shared/*.jade" ]
          dest: "app/compile"
          expand: true
          ext: ".html"
        ]

gulpfile example

    	gulp.task 'jade', ->
			  gulp.src config.paths.jade.src
			  .pipe plugins.jade
			    pretty: true
			  .pipe gulp.dest config.paths.jade.compile

.pipe()

.pipe() is just a function that takes a readable source stream src and hooks the output to a destination writable stream dst

	    	a.pipe(b);
				b.pipe(c);
				c.pipe(d);
			
    		a.pipe(b).pipe(c).pipe(d);
